Advanced Post Submission

If you're a technical person and are not faint of heart, you can submit a blog post with the formatting and any assets you want. You'll need the following tools:

  • git - This site is stored and served out of a git repository
  • A Github account
  • Ruby with the bundler gem installed (optional)
  • Read up on jekyll to become familiar with how this blog is setup.

Once you are ready, do the following:

  • Fork this repository: https://github.com/bratta/geekwriters. This contains all the pages for the blog.
  • Clone your new fork of the geekwriters repository:
    {% highlight bash %} $ git clone git@github.com:yourgithubaccount/geekwriters.git $ cd geekwriters {% endhighlight %}
  • If you have ruby installed, you can get the requirements for jekyll by doing the following:
    {% highlight bash %} $ bundle install {% endhighlight %}
  • You can create pages in the _posts directory using either Markdown syntax, Textile, or HTML. The name of your file should look something like this: YYYY-MM-DD-title.MARKUP, where the first part is the date for your post, the title is the title of your post, and MARKUP is the markup language of your choice (either markdown, textile, or html).
  • You will need to put some metadata at the front of your post. It looks like this: {% highlight yaml %} --- layout: post title: This is a test! author: tim comments: true --- {% endhighlight %} Don't change "layout", but do change the title, the author, and remove the comments line if you do not want to enable commenting on your post. After the final dashed line, you can put the text of your post using the markup language of your choice. See the existing documents in the _posts directory for more detail.
  • If you have ruby installed and ran the bundler step above, instead of creating the file yourself, you can do this:
    {% highlight bash %} $ rake newpost {% endhighlight %} This will ask you for some basic information and create the file for you (assuming Markdown syntax) and open up the new file in your editor. Then all you have to do is write your content, and save the file!
  • When you are done editing, you will need to commit your changes to git and push them up to Github:
    {% highlight bash %} $ git add . $ git commit -m "Adding my new post" $ git push origin master {% endhighlight %}
  • As soon as your changes are pushed up to Github, then follow the Github guide to submitting a pull request. This will notify me of your change and I will add your post to the site by merging in the changes.

Yes, this seems like a lot of work to submit a blog post. But once you get going it is really not bad at all.
